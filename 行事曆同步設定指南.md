# 數辦公用行事曆 — 自動同步設定指南

主管在系統按下「核准 / 核定」後，系統會自動把該筆請假或加班寫入**數辦公用 Google 行事曆**。

本文件帶你完成一次性設定。**程式碼已經寫好**，你只需要做以下「設定」與「填環境變數」。

---

## 運作原理（一句話）

前端按核准 → Firestore 記錄狀態變「核准」→ 呼叫後端 `/api/sync-calendar`
→ 後端用「服務帳戶」確認狀態確實已核准 → 寫入公用行事曆。

> 安全性：後端會自己去 Firestore 驗證狀態，前端無法偽造任意行事曆事件。私鑰只放 Vercel 環境變數，不進前端、不進 GitHub。

---

## 步驟 1：建立服務帳戶並下載私鑰

1. 進入 [Google Cloud Console](https://console.cloud.google.com/)，左上角專案選 **yunlin-digital-travel**（與 Firebase 同一個專案）。
2. 搜尋並進入「**API 和服務 → 程式庫**」，搜尋 **Google Calendar API**，按 **啟用**。
3. 進入「**IAM 與管理 → 服務帳戶**」→「**建立服務帳戶**」。
   - 名稱例如 `calendar-sync`。建立後會得到一個 email，例如
     `calendar-sync@yunlin-digital-travel.iam.gserviceaccount.com`（**等一下會用到，先複製起來**）。
4. 該服務帳戶 →「**金鑰**」分頁 →「**新增金鑰 → 建立新的金鑰 → JSON**」，下載 JSON 檔。
   - ⚠️ 這個 JSON 是機密，**不要放進專案資料夾、不要上傳 GitHub**。

## 步驟 2：讓服務帳戶能讀 Firestore

1. 在「**IAM 與管理 → IAM**」找到剛剛的服務帳戶（或按「授予存取權」貼上它的 email）。
2. 角色選 **Cloud Datastore 使用者**（Cloud Datastore User），儲存。
   - 這讓後端能讀取請假/加班記錄來驗證狀態。

## 步驟 3：把公用行事曆分享給服務帳戶

1. 打開 [Google 行事曆](https://calendar.google.com/)，找到「數辦公用行事曆」。
2. 該行事曆 →「**設定和共用**」。
3. 「**與特定使用者或群組共用**」→ 新增 **步驟 1 那個服務帳戶 email**，
   權限選「**變更活動**」（Make changes to events）。
4. 在同一頁往下找「**整合行事曆**」→ 複製「**行事曆 ID**」
   （長得像 `xxxx@group.calendar.google.com`，**等一下要填**）。

## 步驟 4：在 Vercel 設定環境變數

進 Vercel → 你的專案 → **Settings → Environment Variables**，新增以下四個
（Environment 都勾 **Production**，建議連 Preview 也勾）：

| 變數名稱 | 值 |
|---|---|
| `GOOGLE_PROJECT_ID` | `yunlin-digital-travel` |
| `GOOGLE_CLIENT_EMAIL` | 服務帳戶 email（步驟 1） |
| `GOOGLE_PRIVATE_KEY` | 打開下載的 JSON，複製 `"private_key"` 的值（含 `-----BEGIN PRIVATE KEY-----` 到 `-----END PRIVATE KEY-----`）整段貼上 |
| `OFFICE_CALENDAR_ID` | 行事曆 ID（步驟 3） |

> `GOOGLE_PRIVATE_KEY` 提醒：直接把 JSON 裡那串（含 `\n`）原封不動貼上即可，程式會自動處理換行。

## 步驟 5：部署

把本次新增/修改的檔案放上 GitHub（Vercel 會自動部署）：
- 新增：`api/sync-calendar.js`、`.env.example`、本指南
- 修改：`package.json`（新增 firebase-admin、googleapis）、`src/App.tsx`、`.gitignore`

> ⚠️ 重要：因為新增了 `package.json` 的套件，部署時 Vercel 會重新安裝。
> 部署完成後請先用測試帳號核准一筆假資料，確認行事曆有出現事件，再對外公告。

---

## 測試與驗證

1. 用一般同仁帳號送出一筆請假。
2. 用主管帳號核准它。
3. 看畫面是否出現「已自動同步至數辦公用行事曆！」的提示。
4. 打開公用行事曆，確認事件出現在正確的日期/時間。
5. 若提示「行事曆自動同步失敗：…」，依照下面排錯。

## 常見錯誤排查

| 訊息 | 原因與處理 |
|---|---|
| 伺服器尚未設定完整環境變數 | Vercel 環境變數沒填齊或拼錯；填好後要 **重新部署** 才生效 |
| `Calendar usage limits exceeded` / 403 | 行事曆沒分享給服務帳戶，或權限不是「變更活動」 |
| `Requested entity was not found` | `OFFICE_CALENDAR_ID` 填錯 |
| `PERMISSION_DENIED`（讀 Firestore） | 服務帳戶少了「Cloud Datastore 使用者」角色（步驟 2） |
| `invalid_grant` / 私鑰錯誤 | `GOOGLE_PRIVATE_KEY` 貼不完整或換行壞掉，重貼一次 |
